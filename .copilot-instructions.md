# City Marin Studio - Project Instructions

**PRIMARY OBJECTIVE**: Replicate the design system from `/design/` folder as production-ready React/TypeScript components in Next.js, maintaining 100% visual and functional accuracy.

---

## 🎯 Core Mission (CRITICAL)

**This project has ONE purpose only:**

> Convert the design system prototype (HTML/CSS in `/design/`) into React/TypeScript components using the SCSS architecture already built in `/styles/`.

**What this means:**
- ✅ Visual design = 100% replica of design files
- ✅ Functionality = Exact match of design interactions
- ✅ Code = Production-ready React + TypeScript
- ❌ NO database
- ❌ NO backend API
- ❌ NO testing framework
- ❌ NO authentication
- ❌ NO forms/submissions
- ❌ NO state management beyond component state

**Success Criteria:** User views Next.js site → looks/works exactly like design system files.

## 🛠️ Build Guidelines

### **Step 1: Analyze Design Component**
1. Open reference file in `/design/preview/`
2. Note:
   - HTML structure (semantic tags)
   - CSS classes applied
   - Hover/active/disabled states
   - Responsive breakpoints
   - Animation details (duration, easing, delay)
   - Dark mode styling

### **Step 2: Create React Component**
1. Create file in `/components/`
2. Use TypeScript with strict typing
3. Follow BEM naming convention from SCSS
4. Import SCSS styles directly
5. Use className for styling
6. Include JSDoc comments

**File Location Pattern:**
```
components/
├── [ComponentName]/
│   ├── [ComponentName].tsx      ← React component
│   └── [ComponentName].test.tsx  ← (SKIP - not needed)
```

### **Step 3: Use SCSS Design System**
```typescript
import styles from './Header.module.scss';

export function Header() {
  return (
    <header className={styles.header}>
      {/* Implementation using .scss classes */}
    </header>
  );
}
```

**Available SCSS:**
- Tokens: `/styles/tokens/` (colors, spacing, typography, motion)
- Components: `/styles/components/` (header, card, button, footer, etc.)
- Utilities: `/styles/utilities/_generated.scss` (flex, grid, spacing, text)
- Mixins: `/styles/tools/_mixins.scss` (respond-to, nightshift, transitions)

### **Step 4: Verify Against Design**
1. Visual comparison with `/design/preview/`
2. Check responsive behavior (xs, sm, md, lg, xl, 2xl)
3. Test dark mode (add `nightshift` class to body in browser dev tools)
4. Verify animations smooth, durations accurate
5. Check hover/focus states

## 📐 Design System Integration

### **Colors**
```typescript
// ✅ USE: Design tokens from SCSS
<div className="bg-black text-white">

// ❌ AVOID: Hardcoded hex values
<div style={{ backgroundColor: '#1b1b1b' }}>
```

### **Spacing**
```typescript
// ✅ USE: Space tokens via gap/padding utilities
<div className="gap-4 p-6 m-8">

// ❌ AVOID: Inline margin/padding
<div style={{ margin: '16px', padding: '24px' }}>
```

### **Typography**
```typescript
// ✅ USE: Type scale classes
<h1 className="type-h1">
<p className="type-body">

// ❌ AVOID: Inline font-size
<p style={{ fontSize: '16px' }}>
```

### **Responsive**
```scss
// ✅ USE: Mobile-first with respond-to mixin
.component {
  width: 100%;
  
  @include respond-to('md') {
    width: 50%;
  }
}

// ❌ AVOID: Desktop-first or hardcoded breakpoints
.component { width: 800px; }
```

### **Dark Mode**
```scss
// ✅ USE: nightshift mixin
.component {
  color: $color-black;
  @include nightshift(color, $color-white);
}

// ❌ AVOID: Direct dark mode selectors
.component { color: black; }
body.nightshift .component { color: white; }
```

### **Motion**
```scss
// ✅ USE: Motion tokens from _motion.scss
.component {
  @include transition(all, $duration-base, $ease-in-out);
  animation: slideUp $duration-slow $ease-out-quad;
}

// ❌ AVOID: Hardcoded durations/easing
animation: slideUp 0.6s cubic-bezier(...);
```

## 📋 File Structure

Each component should follow:

```typescript
'use client'; // if using hooks

import type { ReactNode } from 'react';
import styles from './ComponentName.module.scss';

/**
 * ComponentName - Brief description
 * 
 * Visual reference: /design/preview/component-name.html
 * Responsive: xs, sm, md, lg, xl, 2xl
 * Dark Mode: Supported (nightshift class)
 * Animation: Yes (0.3s ease-in-out)
 */

interface ComponentNameProps {
  /** Description of prop */
  children?: ReactNode;
  /** Description of prop */
  variant?: 'primary' | 'secondary';
  /** Mobile/tablet/desktop specific behavior */
  responsive?: boolean;
}

export function ComponentName({
  children,
  variant = 'primary',
  responsive = true,
}: ComponentNameProps) {
  return (
    <div className={`${styles.component} ${styles[variant]}`}>
      {children}
    </div>
  );
}
```
